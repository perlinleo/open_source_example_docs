## üõ† How to Contribute

### 1. Discuss Before Submitting
Before working on any change, please:

- Open a **Discussion** or **Issue**
- Describe the improvement or fix
- Wait for approval from the maintainer (@perlinleo)

Unapproved, large PRs may be rejected.

---

### 2. Forking and Branching Rules

You may create a **temporary fork/branch** *only* for the purpose of submitting a pull request to this repository.

You may **NOT**:

- Publish your fork
- Distribute it
- Use it as a basis for a new project
- Create derivative works outside this repository

Any forks must remain private or deleted after your PR is merged or closed.

---

### 3. Submitting a Pull Request

When submitting a PR:

- Use clear, descriptive commit messages
- Keep changes small and focused
- Run tests (if applicable)
- Follow the repository‚Äôs style and conventions
- Ensure no unrelated changes are included
- Ensure the PR description uses the provided PR Template

The maintainer (@perlinleo) has full discretion to accept or reject any contribution.
